# SRE Playbook

## SLO/SLI/SLA 框架

### SLI (Service Level Indicator)
可量化的服务度量指标：
- **可用性:** 成功请求 / 总请求
- **延迟:** P50, P95, P99 响应时间
- **吞吐量:** QPS / TPS
- **错误率:** 5xx 错误 / 总请求
- **饱和度:** CPU/内存/队列深度

### SLO (Service Level Objective)
SLI 的目标值：
- API 可用性: 99.9% (月度)
- P99 延迟: < 200ms
- 错误率: < 0.1%

### SLA (Service Level Agreement)
SLO 违约的商业后果：
- 99.9% → 月度停机 < 43 分钟
- 99.95% → 月度停机 < 22 分钟
- 99.99% → 月度停机 < 4.3 分钟

## 故障响应流程

### 严重级别

| 级别 | 定义 | 响应时间 | 通知范围 |
|------|------|----------|----------|
| P0 | 全站不可用 | 5 分钟 | 全员 + 管理层 |
| P1 | 核心功能受损 | 15 分钟 | 团队 + 主管 |
| P2 | 非核心功能受损 | 1 小时 | On-call |
| P3 | 性能降级 | 下一工作日 | 相关人员 |

### 故障处理步骤

1. **检测** — 监控告警触发
2. **确认** — 确认影响范围
3. **沟通** — 更新状态页、通知相关方
4. **止损** — 回滚/降级/扩容
5. **根因** — 定位根因
6. **修复** — 永久修复
7. **复盘** — Postmortem 文档

## 容量规划

### 指标
- **当前容量:** 当前最大处理能力
- **峰值负载:** 历史最高负载
- **安全边际:** 峰值的 1.5-2x
- **增长预测:** 未来 3-6 月增长趋势

### 扩容触发条件
- CPU 持续 > 70%
- 内存持续 > 80%
- 队列深度 > 阈值
- P99 延迟 > SLO

## 变更管理

### 发布策略
- **蓝绿部署:** 两套环境切换
- **金丝雀发布:** 小比例流量验证
- **灰度发布:** 逐步扩大流量
- **功能开关:** 代码级控制
