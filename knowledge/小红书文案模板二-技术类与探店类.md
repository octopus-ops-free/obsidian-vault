## 一、技术/数码类

### 核心逻辑
技术类笔记的挑战在于"把专业语言翻译成大众能懂的表达"。用户不关心你的代码多优雅，只关心"这东西能帮我解决什么问题"。

### 5种高流量内容方向

#### 1. AI工具效率提升型（涨粉最快）
**标题公式：** AI工具 + 效率提升 + 人群
**案例：**
- "5款AI工具帮我把工作效率提升10倍！加班狗直接变摸鱼达人"
- "用ChatGPT+Cursor写代码，5分钟搞定电商后台，程序员看完哭了"
- "用Midjourney做小红书头像，每单29.9元，1个月赚12000"

**正文结构：**
```
痛点引入（手动操作太慢/太累）
→ 工具介绍（名称+功能+价格）
→ 使用前后对比（时间/效果）
→ 具体操作步骤
→ 效果展示（截图/数据）
→ 获取方式
```

#### 2. 编程教程保姆级型（收藏率最高）
**标题公式：** 零基础/小白 + 学习内容 + 效果承诺
**案例：**
- "零基础转行程序员的真实经历｜从月薪3k到2w"
- "Python自动化处理Excel报表，10行代码搞定"
- "1024程序员节｜我靠这5个GitHub项目涨粉1万+"

**正文结构：**
```
学习背景（为什么学/踩过什么坑）
→ 技术选型（为什么选这个语言/框架）
→ 分步骤代码示例（配截图，不要太长）
→ 常见报错+解决方法
→ 学习资源推荐
→ 互动引导（"你们想学什么？评论区告诉我"）
```

#### 3. 数码好物推荐型（变现能力最强）
**标题公式：** 身份 + 品类 + 数量/效果
**案例：**
- "程序员桌搭｜这5件好物让我的效率翻倍"
- "全栈开发者的2025装备清单｜总花费不到3000"
- "自由职业3年，这台MacBook陪我赚了50万"

**正文结构：**
```
使用场景说明
→ 每件产品的FAB（属性→优势→益处）
→ 真实使用感受（要有小缺点，增加可信度）
→ 价格+购买渠道
→ 适合什么人群
```

#### 4. 职场经验分享型（互动率最高）
**标题公式：** 身份 + 经验/内幕 + 数字
**案例：**
- "大厂3年，我总结了程序员最该学的5个软技能"
- "从面试官角度：90%的简历都死在这3个问题上"
- "字节跳动面试官告诉你：简历这样写，面试邀约翻倍"

**正文结构：**
```
身份背书（"作为XX年的从业者"）
→ 核心观点（3-5个，每个配案例说明）
→ 实用建议（普通人如何应对）
→ 互动引导（"你们遇到过哪些坑？"）
```

#### 5. 副业变现型（转发率最高）
**标题公式：** 方法 + 收入 + 时间
**案例：**
- "用AI做小红书，0成本月入8000的实操路径"
- "程序员下班后做副业，这3个方向最靠谱"
- "自由职业第一年｜我靠接外包赚了20万"

**正文结构：**
```
个人经历引入（"我是怎么开始的"）
→ 变现路径拆解（分步骤）
→ 收入数据（真实截图，增强可信度）
→ 时间投入说明
→ 避坑提醒
→ 适合什么人做
```

### 技术类封面设计要点
- **代码截图型** — 关键代码高亮，配大字标题
- **工具界面型** — 效果截图+使用前后对比
- **大字报型** — 核心结论用大字展示（"效率提升10倍"）
- **清单型** — 罗列要点，信息密度高

### 技术类发布技巧
- 最佳时间：工作日 12:00-13:30、20:00-22:00
- 标签：#程序员 #效率工具 #AI工具 #编程教程 #副业
- 垂直定位很重要：选一个细分领域深耕，不要什么都做
- 用类比和场景化语言，避免专业术语堆砌

## 二、探店/美食类

### 核心逻辑
探店笔记的核心是"真实体验感"，用户想知道"这家店值不值得我专门跑一趟"。要让人看文字就仿佛坐在餐厅里。

### 5种高流量内容方向

#### 1. 宝藏店铺发现型（流量最大）
**标题公式：** 位置线索 + 惊喜感 + 情绪词
**案例：**
- "藏在巷子里的小馆子，被这碗牛肉面香到舔碗"
- "在商场角落藏了三年，中午12点去居然要排队"
- "居民楼下开了10年的苍蝇馆子，本地人才知道"

**正文结构：**
```
发现经过（"下班路过/朋友推荐/偶然发现"）
→ 环境描述（风格+氛围+拍照打卡点）
→ 菜品描述（用感官词：脆/嫩/鲜/香，配具体口感细节）
→ 价格信息（人均消费+性价比评价）
→ 实用Tips（停车/排队/最佳位置/推荐菜品）
→ 互动引导（"你们觉得这家跟XX比哪个更值得去？"）
```

#### 2. 合集推荐型（收藏率最高）
**标题公式：** 城市/品类 + 数量 + 限定词
**案例：**
- "成都本地人私藏的10家小馆子，第3家绝了"
- "上海适合约会的8家氛围感餐厅｜人均100起"
- "打工人午餐不重样！写字楼周边5家宝藏简餐"

**正文结构：**
```
筛选标准说明（"为什么选这几家"）
→ 每家店：店名+招牌菜+人均+一句话点评
→ 按场景/价位/风格分类
→ 综合推荐（"首选哪家"）
→ 收藏引导
```

#### 3. 排雷避坑型（点击率最高）
**标题公式：** 警示词 + 城市/品类
**案例：**
- "去三亚千万别做这5件事！后悔没早知道"
- "网红店排雷｜这3家真的不值得排队2小时"
- "本地人告诉你：来XX千万别去这些地方吃饭"

**正文结构：**
```
踩坑经历（制造紧张感）
→ 逐一列坑（每个坑+后果+正确替代方案）
→ 总结正确做法
→ 正确店铺推荐
```

#### 4. 美食教程型（长期价值最高）
**标题公式：** 难度词 + 菜品 + 效果
**案例：**
- "0失败教程！在家复刻茶颜悦色同款奶茶"
- "平底锅就能做的5道神仙早餐，10分钟搞定"
- "这个麻辣拌配方，知道的人越多，倒闭的店越多"

**正文结构：**
```
成品展示（让人流口水的图）
→ 食材清单（精确到克数）
→ 步骤分解（每步配图+关键技巧）
→ 常见失败原因+解决办法
→ 口感描述
→ 收藏引导（"收藏=学会"）
```

#### 5. 旅行+探店结合型（覆盖面最广）
**标题公式：** 地点 + 天数 + 预算/体验
**案例：**
- "人均500玩转厦门｜3天2夜省钱攻略+美食地图"
- "潮汕3天吃了20家｜本地人带路不踩雷"
- "云南旅行最全美食攻略｜这15家店必须打卡"

**正文结构：**
```
行程概览（天数+路线+预算）
→ 每天的美食打卡（配图+店名+地址+推荐菜）
→ 交通住宿tips
→ 费用明细
→ 避坑提醒
→ 收藏引导
```

### 探店封面设计要点
- **食物特写型** — 突出食材质感，自然光最佳，侧光/逆光勾勒轮廓
- **环境氛围型** — 适合约会/聚会场景，暖色调
- **对比型** — 多道菜品平铺 vs 单道菜特写
- **大字报型** — "人均30"、"排队2小时" 等关键信息压字

### 探店拍摄技巧
- 自然光是最佳滤镜，选靠窗位置
- 45度角特写突出食物细节
- 俯拍适合展示全貌
- 适当手部入镜（拿取食物），增加代入感
- 后期调色降低对比度，呈现柔和质感

### 探店文案写作要素
| 要素 | 写什么 | 案例 |
|------|--------|------|
| 环境 | 风格+氛围+拍照点 | "ins风适合自拍，复古风适合氛围大片" |
| 菜品 | 口感层次+特殊食材+细节 | "外皮脆到咬下去能听到咔嚓声，肉汁顺着指缝流下来" |
| 价格 | 人均+性价比+优惠 | "人均¥80，周三女生同行第二份半价" |
| 服务 | 态度+特色+注意事项 | "服务一般但味道没话说" |
| 实用信息 | 地址+停车+排队+最佳时间 | "建议避开饭点，赶时间直接点招牌套餐" |

### 探店标签组合
- 地域标签：#上海美食 #XX区探店 #陆家嘴下午茶
- 品类标签：#日料 #火锅 #咖啡 #甜品
- 场景标签：#约会餐厅 #拍照打卡 #周末去哪儿
- 通用标签：#探店 #美食探店 #吃货日常

### 探店最佳发布时间
- 午餐前：11:00-13:00（触发午餐决策）
- 晚餐前：17:00-19:00（触发晚餐决策）
- 深夜：22:00-24:00（深夜放毒，饥饿营销）

## 三、通用技巧补充

### 真实感是杀手锏
- 探店时说"服务一般，但味道没话说"
- 测评时说"好吃但热量太高，减肥慎入"
- 教程时说"我这次糖放多了，你们按我说的量来"
- 用户要的是"一个活生生的分享者"，不是冷冰冰的广告

### 技术+美食的交叉选题（差异化竞争）
- "用Python分析了10000条小红书探店数据，发现这5个规律"
- "程序员的周末探店Vlog｜用数据分析找餐厅"
- "用AI生成了一周食谱，照着做居然没翻车"
